# QC Oil Calendar Spread

A QuantConnect (Lean) mean-reversion algorithm that trades the WTI crude-oil
calendar spread (front month vs second month), enhanced with Cushing inventory
and FRED economic signals loaded from **pre-downloaded CSV files**.

No external API calls are made during backtesting.

---

## Quick Start

### 1 – Install local dependencies (data download only)

```bash
pip install requests   # only needed to run data_downloader.py
```

### 2 – Obtain API keys

| Service | URL                                    | Free tier |
|---------|----------------------------------------|-----------|
| EIA     | https://www.eia.gov/opendata/register/ | ✅         |
| FRED    | https://fred.stlouisfed.org/docs/api/  | ✅         |

### 3 – Set environment variables

```bash
export EIA_API_KEY="your_eia_key_here"
export FRED_API_KEY="your_fred_key_here"
```

Alternatively, create a `secrets.env` file in the project root
(**never commit this file** – it is gitignored):

```
EIA_API_KEY=your_eia_key_here
FRED_API_KEY=your_fred_key_here
```

### 4 – Download the data

```bash
python data_downloader.py
```

This writes:
* `data/cushing_inventory.csv` – weekly Cushing crude-oil stocks (EIA)
* `data/fred_data.csv` – selected FRED series (WTI, Brent, 10Y Treasury)

### 5 – Upload to QuantConnect

1. Open your QuantConnect project.
2. Upload all `.py` files and the entire `data/` directory.
3. Set the main file to `main_algorithm.py`.
4. Run the backtest.

> **Tip:** If the CSV files are not found, `data_manager.py` automatically
> falls back to seasonally-derived simulated data so the algorithm still
> executes (useful for smoke-testing the logic).

## Configuration

Edit `config.py` to adjust:

| Parameter             | Default | Description                                  |
|-----------------------|---------|----------------------------------------------|
| `FRONT_MONTH_TICKER`  | `CL1`   | Front-month futures symbol                   |
| `BACK_MONTH_TICKER`   | `CL2`   | Second-month futures symbol                  |
| `ENTRY_Z_SCORE`       | `1.5`   | Z-score threshold to enter a position        |
| `EXIT_Z_SCORE`        | `0.5`   | Z-score threshold to exit a position         |
| `LOOKBACK_DAYS`       | `252`   | Rolling window for spread z-score            |
| `MAX_POSITION_SIZE`   | `0.25`  | Max fraction of portfolio per leg            |
| `INVENTORY_LOOKBACK_WEEKS` | `52` | Weeks of history for inventory z-score    |

## Why CSV-based data?

QuantConnect's backtester runs in a sandboxed environment and **cannot** make
outbound HTTP requests to external APIs such as EIA or FRED.  Attempting to do
so causes the backtest to crash or hang with rate-limiting errors.

By downloading data once with `data_downloader.py` and loading it from CSVs
at backtest time, we get:

* ✅ Reliable execution – no network dependencies at run-time
* ✅ Reproducible backtests – same data every time
* ✅ Lower QCC consumption – no network overhead
* ✅ Faster backtests – data is read from memory after the initial load
* ✅ Proper gap handling – nearest-date fallback in `FundamentalDataManager`
